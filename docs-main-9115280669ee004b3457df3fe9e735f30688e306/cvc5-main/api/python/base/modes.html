class cvc5.LearnedLitType(*values)ÔÉÅ

The LearnedLitType enum

CONSTANT_PROPÔÉÅ

An internal literal that can be made into a constant propagation for an input term.

In particular, literals in this category are of the form (= t c) where c is a constant, the preprocessed set of input formulas contains the term t, but not the literal (= t c).

INPUTÔÉÅ

A literal from the preprocessed set of input formulas that does not occur at top-level after preprocessing.

Typically), this is the most interesting category of literals to learn.

INTERNALÔÉÅ

Any internal literal that does not fall into the above categories.

PREPROCESSÔÉÅ

A top-level literal (unit clause) from the preprocessed set of input formulas.

PREPROCESS_SOLVEDÔÉÅ

An equality that was turned into a substitution during preprocessing.

In particular, literals in this category are of the form (= x t) where x does not occur in t.

SOLVABLEÔÉÅ

An internal literal that is solvable for an input variable.

In particular, literals in this category are of the form (= x t) where x does not occur in t, the preprocessed set of input formulas contains the term x, but not the literal (= x t).

Note that solvable literals can be turned into substitutions during preprocessing.

UNKNOWNÔÉÅ

Special case for when produce-learned-literals is not set.

class cvc5.ProofComponent(*values)ÔÉÅ

The ProofComponent enum

FULLÔÉÅ

A proof of false whose free assumptions are a subset of the input formulas F1), ‚Ä¶ Fm.

Only valid immediately after an unsat response.

PREPROCESSÔÉÅ

Proofs of Gu1 ‚Ä¶ Gun whose free assumptions are Fu1, ‚Ä¶ Fum, where: - Gu1, ‚Ä¶ Gun are clauses corresponding to input formulas used in the SAT proof, - Fu1, ‚Ä¶ Fum is the subset of the input formulas that are used in the SAT proof (i.e. the unsat core).

Note that Gu1 ‚Ä¶ Gun are clauses that are added to the SAT solver before its main search.

Only valid immediately after an unsat response.

RAW_PREPROCESSÔÉÅ

Proofs of G1 ‚Ä¶ Gn whose free assumptions are a subset of F1, ‚Ä¶ Fm, where: - G1, ‚Ä¶ Gn are the preprocessed input formulas, - F1, ‚Ä¶ Fm are the input formulas.

Note that G1 ‚Ä¶ Gn may be arbitrary formulas, not necessarily clauses.

SATÔÉÅ

A proof of false whose free assumptions are Gu1, ‚Ä¶ Gun, L1 ‚Ä¶ Lk, where: - Gu1, ‚Ä¶ Gun, is a set of clauses corresponding to input formulas, - L1, ‚Ä¶, Lk is a set of clauses corresponding to theory lemmas.

Only valid immediately after an unsat response.

THEORY_LEMMASÔÉÅ

Proofs of L1 ‚Ä¶ Lk where: - L1, ‚Ä¶, Lk are clauses corresponding to theory lemmas used in the SAT proof.

In contrast to proofs given for preprocess, L1 ‚Ä¶ Lk are clauses that are added to the SAT solver after its main search.

Only valid immediately after an unsat response.

class cvc5.FindSynthTarget(*values)ÔÉÅ

The FindSynthTarget enum

ENUMÔÉÅ

Find the next term in the enumeration of the target grammar.

QUERYÔÉÅ

Find a query over the given grammar. If the given grammar generates terms that are not Boolean, we consider equalities over terms from the given grammar.

The algorithm for determining which queries to generate is configured by ‚Äìsygus-query-gen=MODE. Queries that are internally solved can be filtered by the option ‚Äìsygus-query-gen-filter-solved.

REWRITEÔÉÅ

Find a pair of terms (t,s) in the target grammar which are equivalent but do not rewrite to the same term in the given rewriter (‚Äìsygus-rewrite=MODE). If so, the equality (= t s) is returned by findSynth.

This can be used to synthesize rewrite rules. Note if the rewriter is set to none (‚Äìsygus-rewrite=none), this indicates a possible rewrite when implementing a rewriter from scratch.

REWRITE_INPUTÔÉÅ

Find a rewrite between pairs of terms (t,s) that are matchable with terms in the input assertions where t and s are equivalent but do not rewrite to the same term in the given rewriter (‚Äìsygus-rewrite=MODE).

This can be used to synthesize rewrite rules that apply to the current problem.

REWRITE_UNSOUNDÔÉÅ

Find a term t in the target grammar which rewrites to a term s that is not equivalent to it. If so, the equality (= t s) is returned by findSynth.

This can be used to test the correctness of the given rewriter. Any returned rewrite indicates an unsoundness in the given rewriter.
